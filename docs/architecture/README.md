# Architecture

This directory documents the design philosophy and structural constraints of the system.

## Document Boundaries

* **[principles.md](./principles.md)** — *why* the system is designed this way (philosophy, tradeoffs)
* **[layers.md](./layers.md)** — *what* the structural organization is (responsibilities, data flow)
* **[boundaries.md](./boundaries.md)** — *who enforces* the rules and what is forbidden

If content could fit in two of these, ask: is it a conviction (principles),
a structural fact (layers), or an enforcement rule (boundaries)?
